# 命名空间
## 创建命名空间
- '''namespace A{}'''
- 命名空间只能全局范围定义
- 命名空间可以嵌套
- 命名空间是开放的，可以随时把新的成员加入已有的命名空间中
- 声明和实现可以分离
- 无名命名空间，意味着命名空间中的标识符只能在本文件内访问，相当于给这个标识符加上了static，使得其可以作为内部连接
- 命名空间别名 namespace shortName = veryLongName;
- 使用using声明或using编译指令会增加命名冲突的可能性。也就是说，如果有名称空间，并在代码中使用作用域解析运算符，则不会出现二义性。

## struct 类型加强
- c++中在结构体内部既可以定义成员变量，又可以定义成员函数
- 在c++中，三目运算符返回的是左值，可以进行赋值

## 左值右值的概念
- ++中可以放在赋值操作符左边的是左值，可以放到赋值操作符右面的是右值。
- 变量即可以当左值，也可以当右值。
- 左值为Lvalue，L代表Location，表示内存可以寻址，可以赋值。
- 右值为Rvalue，R代表Read,就是可以知道它的值。
- 比如:int temp = 10; temp在内存中有地址，10没有，但是可以Read到它的值。

## const
- const修饰的变量是常量，不能被修改，但是可以被赋值。
- const修饰的指针是指向常量的指针，不能通过指针修改指向的值，但是可以修改指针的值。
- 在c++中，一个const不必创建内存空间，而在c中，一个const总是需要一块内存空间。

## 引用
- 引用是一个别名，引用必须初始化，引用一旦初始化后，就不能再改变了。
- 引用的本质是指针，引用的本质是指针，引用的本质是指针。

## 内联函数
- inline voidfunction（int a){retuen a++;};
- 函数体和声明需要放在一起，不能分离。
- 空间换时间
- 任何在类内部定义的函数自动成为内联函数。

##函数的默认参数
- 函数的默认参数从左向右，如果一个参数设置了默认参数，那么这个参数之后的参数都必须设置默认参数。
- 如果函数声明和函数定义分开写，函数声明和函数定义不能同时设置默认参数。

## 占位参数
- void TestFunc01(int a,int b,int)
- void TestFunc01(int a,int b,int = 10)
- 占位参数只能有一个，且必须放在参数列表的最后。
- 函数内部无法使用占位参数

##函数重载
- 函数名相同，参数列表不同，返回值可以相同也可以不同。
- 同一个作用域
- 参数个数不同
- 参数类型不同
- 参数顺序不同
- 返回值不作为函数重载的条件

## extern "C"
- 主要作用就是为了实现c++代码能够调用其他c语言代码
- 加上extern "C"后，这部分代码编译器按c语言的方式进行编译和链接，而不是按c++的方式
